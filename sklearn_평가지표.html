01
confusion_matrix() 분류 기본

모든 분류 평가지표의 기반. 예측값과 실제값의 교차표(Cross Table)로 TP·FP·FN·TN을 한눈에 확인.

이진분류 다중분류 sklearn.metrics

TP: 양성→양성 예측 ✓ 정답

FP: 음성→양성 예측 ✗ 오류 (Type I)
FN: 양성→음성 예측 ✗ 오류 (Type II)
파생 지표수식의미범위

specificityTN / (TN+FP)음성 감지율 (True Negative Rate)0~1 ↑
Accuracy
Accuracy = (TP + TN) / (TP + FP + FN + TN)
Precision · Recall · F1
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1 = 2 × (Precision × Recall) / (Precision + Recall)
✅ Precision 중시
  • 스팸 메일 분류 (FP 비용↑)
  • 불량품 알람 (과잉 알람 방지)
  • 금융 사기 FP 최소화
🔴 Recall 중시
  • 암 진단 (FN 비용↑, 미탐지 위험)
  • 화재·이상 탐지 시스템
  • 의료 스크리닝
Precision-Recall 트레이드오프임계값(threshold) 낮추면 Recall↑ Precision↓. 높이면 반대
클래스 불균형 시 주의Accuracy 99%지만 Recall 0일 수 있음. F1·AUC 필수 확인
다중분류 평균 방식macro: 클래스 균등 / weighted: 샘플 수 가중 / micro: 전체 합산
Fbeta 지표β>1 → Recall 강조 / β<1 → Precision 강조 (β=1이 F1)
Confusion Matrix + 파생 지표 전체
1모델 학습 2confusion_matrix 3ConfusionMatrixDisplay 4파생 지표 계산
Python
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import (
    confusion_matrix, ConfusionMatrixDisplay,
    accuracy_score, precision_score, recall_score,
    f1_score, fbeta_score
)
import matplotlib.pyplot as plt

# 데이터 준비
X, y = load_breast_cancer(return_X_y=True)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=.2, random_state=42,
                                            stratify=y)

# 모델 학습
clf = RandomForestClassifier(n_estimators=100, random_state=42)
clf.fit(X_tr, y_tr)
y_pred = clf.predict(X_te)

# ─── 혼동 행렬 ───
cm = confusion_matrix(y_te, y_pred)
print("Confusion Matrix:\n", cm)
# [[TN, FP],
#  [FN, TP]]

# ConfusionMatrixDisplay 시각화
disp = ConfusionMatrixDisplay(confusion_matrix=cm,
                               display_labels=['악성(0)', '양성(1)'])
fig, ax = plt.subplots(figsize=(5, 4))
disp.plot(ax=ax, colorbar=False, cmap='Blues')
ax.set_title('Confusion Matrix — Breast Cancer')
plt.tight_layout(); plt.show()

# ─── 파생 지표 ───
tn, fp, fn, tp = cm.ravel()
print(f"\n[직접 계산]")
print(f"  Accuracy   : {(tp+tn)/(tp+tn+fp+fn):.4f}")
print(f"  Precision  : {tp/(tp+fp):.4f}")
print(f"  Recall     : {tp/(tp+fn):.4f}")
print(f"  Specificity: {tn/(tn+fp):.4f}   ← TNR")

print(f"\n[sklearn 함수]")
print(f"  accuracy   : {accuracy_score(y_te, y_pred):.4f}")
print(f"  precision  : {precision_score(y_te, y_pred):.4f}")
print(f"  recall     : {recall_score(y_te, y_pred):.4f}")
print(f"  f1         : {f1_score(y_te, y_pred):.4f}")
print(f"  f2 (β=2)   : {fbeta_score(y_te, y_pred, beta=2):.4f}  ← recall 중시")
📊 결과 해석
  • Breast Cancer 데이터에서 RF는 Accuracy ≈ 0.965, Recall ≈ 0.972 수준
  • FN(암인데 음성 판정)이 더 위험 → Recall을 우선 지표로 사용
  • fbeta_score(β=2) → Recall을 Precision보다 2배 가중
  • cm.ravel()로 TN/FP/FN/TP를 한 번에 언팩
💡 팁: normalize='true' 옵션으로 비율 표시 가능: confusion_matrix(y_te, y_pred, normalize='true')
02
classification_report() 종합 리포트

클래스별 Precision·Recall·F1·Support를 한 번에 출력. 다중분류에서 가장 많이 사용하는 종합 평가 함수.

이진분류 다중분류 불균형 데이터 필수
출력 항목설명비고
precision클래스별 정밀도 (FP 최소화)↑ 좋음
recall클래스별 재현율 (FN 최소화)↑ 좋음
f1-scoreP·R 조화평균↑ 좋음
support해당 클래스 실제 샘플 수참고
macro avg클래스별 단순 평균 (균등 가중)소수 클래스 중요 시
weighted avgsupport 가중 평균일반적 권장
Macro vs Weighted vs Micro Average
macro_F1 = (F1_cls0 + F1_cls1 + … + F1_clsN) / N
weighted_F1 = Σ(F1_i × support_i) / Σsupport_i
micro_F1 = F1(TP_total, FP_total, FN_total)
macro avg모든 클래스를 동등하게 취급. 소수 클래스 성능이 중요한 경우
weighted avg샘플 수 비례 가중. 불균형 데이터에서 전체 성능 요약 시
micro avg전체 TP/FP/FN 합산. 다중 레이블 분류에서 주로 사용
zero_division예측 샘플 없는 클래스: 0 또는 1로 처리 (경고 제어 가능)
classification_report — 다중분류 (Iris)
1학습·예측 2report 출력 3dict 변환 활용
Python
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.metrics import classification_report
import pandas as pd

X, y = load_iris(return_X_y=True, as_frame=True)
target_names = ['setosa', 'versicolor', 'virginica']

X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=.25,
                                            stratify=y, random_state=42)
clf = GradientBoostingClassifier(random_state=42)
clf.fit(X_tr, y_tr)
y_pred = clf.predict(X_te)

# ─── 텍스트 출력 ───
print(classification_report(y_te, y_pred, target_names=target_names))
# Output:
#               precision    recall  f1-score   support
#      setosa       1.00      1.00      1.00        13
#  versicolor       0.93      0.93      0.93        14
#   virginica       0.92      0.92      0.92        11
#    accuracy                           0.95        38
#   macro avg       0.95      0.95      0.95        38
# weighted avg       0.95      0.95      0.95        38

# ─── dict로 변환 → DataFrame ───
report_dict = classification_report(y_te, y_pred,
                                     target_names=target_names,
                                     output_dict=True)
df_report = pd.DataFrame(report_dict).T.round(3)
print("\n[DataFrame 형태]\n", df_report)

# ─── 특정 지표만 추출 ───
print(f"\nmacro F1   : {report_dict['macro avg']['f1-score']:.4f}")
print(f"weighted F1: {report_dict['weighted avg']['f1-score']:.4f}")
📊 결과 해석
  • output_dict=True로 받으면 pandas DataFrame으로 시각화 가능
  • support가 작은 클래스(virginica=11)는 F1 변동이 크므로 주의
  • macro avg < weighted avg이면 소수 클래스 성능이 낮다는 신호
  • 클래스별 F1이 크게 차이나면 → 불균형 처리(오버샘플링 등) 검토
💡 팁: zero_division=0 파라미터로 예측이 없는 클래스의 경고를 억제할 수 있음

AUC = ∫₀¹ TPR(FPR) dFPR ∈ [0, 1]

AUC = 1.0완벽 분류기. 곡선이 좌상단 꼭지점 통과

Python
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.svm import SVC
from sklearn.metrics import roc_auc_score, roc_curve, RocCurveDisplay
import matplotlib.pyplot as plt
import numpy as np

X, y = load_breast_cancer(return_X_y=True)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=.2,
                                            stratify=y, random_state=42)
models = {
    'Logistic'  : Pipeline([('sc', StandardScaler()),
                             ('clf', LogisticRegression(max_iter=500))]),
    'RF'        : RandomForestClassifier(n_estimators=100, random_state=42),
    'GBM'       : GradientBoostingClassifier(random_state=42),
    'SVM(RBF)'  : Pipeline([('sc', StandardScaler()),
                             ('clf', SVC(probability=True))]),
}

fig, ax = plt.subplots(figsize=(7, 5))
for name, model in models.items():
    model.fit(X_tr, y_tr)
    y_prob = model.predict_proba(X_te)[:, 1]
    auc = roc_auc_score(y_te, y_prob)
    fpr, tpr, _ = roc_curve(y_te, y_prob)
    ax.plot(fpr, tpr, lw=2, label=f'{name}  (AUC={auc:.3f})')

ax.plot([0,1],[0,1],'k--', lw=1, alpha=.5, label='Random (AUC=0.500)')
ax.set(xlabel='FPR (False Positive Rate)',
       ylabel='TPR (Recall)',
       title='ROC Curve Comparison — Breast Cancer')
ax.legend(loc='lower right', fontsize=10)
plt.tight_layout(); plt.show()

# ─── 최적 임계값 탐색 (Youden's J) ───
model_best = GradientBoostingClassifier(random_state=42)
model_best.fit(X_tr, y_tr)
y_prob_best = model_best.predict_proba(X_te)[:, 1]
fpr, tpr, thresholds = roc_curve(y_te, y_prob_best)
j_stat = tpr - fpr
best_idx = np.argmax(j_stat)
print(f"최적 임계값: {thresholds[best_idx]:.4f}")
print(f"→ TPR={tpr[best_idx]:.3f}, FPR={fpr[best_idx]:.3f}")
📊 결과 해석
  • GBM·RF 모두 AUC ≈ 0.995 — Breast Cancer는 고품질 데이터
  • Youden's J = TPR - FPR 최대화 지점이 최적 임계값
  • 기본 임계값 0.5가 아닌 최적값 적용 시 Recall·Precision 균형 개선
  • SVC는 probability=True 필수 (platt scaling 내부 사용)
💡 팁: 다중분류 AUC → roc_auc_score(y_te, y_prob, multi_class='ovr', average='weighted')

average_precision_score() · PrecisionRecallDisplay PR Curve

Precision vs Recall 곡선. 극단적 클래스 불균형(양성 비율 <5%)에서 ROC-AUC보다 민감한 평가 지표.

이진분류 극단적 불균형 데이터 AP Score
✅ PR Curve 사용
  • 양성 비율 <10% 극단 불균형
  • 사기 탐지, 희귀 질환 진단
  • 정보 검색 시스템 평가
🔴 ROC-AUC 사용
  • 균형 잡힌 데이터셋
  • FPR 패널티가 중요한 경우
  • 다중분류 종합 평가
지표범위의미
average_precision0~1 ↑PR 곡선 아래 면적 (AP Score). 랜덤 기준선 = 양성 비율

Python
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.metrics import (average_precision_score,
                              precision_recall_curve,
                              PrecisionRecallDisplay)
import matplotlib.pyplot as plt

# ─── 불균형 데이터 생성 (양성 5%) ───
X, y = make_classification(n_samples=5000, n_features=20,
                            weights=[0.95, 0.05],   # 95:5 불균형
                            random_state=42)
print(f"양성 비율: {y.mean()*100:.1f}%")

X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=.2,
                                            stratify=y, random_state=42)

models = {
    'Logistic': LogisticRegression(class_weight='balanced', max_iter=500),
    'RF'      : RandomForestClassifier(class_weight='balanced', random_state=42),
    'GBM'     : GradientBoostingClassifier(random_state=42),
}

fig, ax = plt.subplots(figsize=(7, 5))
for name, model in models.items():
    model.fit(X_tr, y_tr)
    y_prob = model.predict_proba(X_te)[:, 1]
    ap = average_precision_score(y_te, y_prob)
    prec, rec, _ = precision_recall_curve(y_te, y_prob)
    ax.plot(rec, prec, lw=2, label=f'{name}  (AP={ap:.3f})')

# 랜덤 기준선 (양성 비율)
baseline = y_te.mean()
ax.axhline(baseline, color='k', ls='--', lw=1,
           label=f'Random baseline ({baseline:.2f})')
ax.set(xlabel='Recall', ylabel='Precision',
       title='Precision-Recall Curve (Imbalanced, Pos=5%)',
       xlim=[0,1], ylim=[0,1])
ax.legend(loc='upper right', fontsize=10)
plt.tight_layout(); plt.show()
📊 결과 해석
  • 불균형 데이터에서 Random baseline = 양성 비율(5%) → AUC 0.5와 다름
  • AP가 높을수록 높은 Recall에서도 Precision 유지됨
  • class_weight='balanced': 불균형 보정 자동 적용
  • GBM은 보통 AP에서 가장 우수한 성능 (PR 곡선 좌상단에 가까움)

MAE · MSE · RMSE · MAPE 오차 지표

회귀 모델의 예측 오차를 측정하는 지표. 이상치 민감도와 해석 용이성에 따라 선택.

회귀 전용 y_true와 동일 단위 낮을수록 좋음
지표수식이상치 민감도해석특징
MAE
Mean Absolute Error		mean(|y-ŷ|)낮음 ✓y단위와 동일직관적, 로버스트
MSE
Mean Squared Error		mean((y-ŷ)²)높음 ✗y단위²미분 가능, 최적화 용이
RMSE
Root MSE		√MSE높음 ✗y단위와 동일MSE + 직관성. 가장 보편적
MAPE
Mean Abs % Error		mean(|y-ŷ|/|y|)×100중간% 단위단위 무관, y=0 시 불정의
MSLE
Mean Squared Log Error		mean((log(y+1)-log(ŷ+1))²)낮음 ✓로그 단위지수적 성장 타겟에 적합
MAE · MSE · RMSE · MAPE
MAE = (1/n) × Σ|yᵢ - ŷᵢ|
MSE = (1/n) × Σ(yᵢ - ŷᵢ)²
RMSE = √MSE
MAPE = (100/n) × Σ|(yᵢ - ŷᵢ) / yᵢ| [%]
✅ MAE 선택
  • 이상치가 많은 데이터
  • 모든 오차를 동등하게 취급
  • 중앙값 기반 예측 평가
✅ RMSE 선택
  • 큰 오차에 패널티 강화
  • 모델 학습 손실 함수
  • 정규분포 가정 오차

r2_score() · Adjusted R² 설명력 지표

모델이 종속변수의 분산을 얼마나 설명하는지 나타내는 지표. 1에 가까울수록 좋고, 0이면 평균 예측과 동일 수준.

회귀 전용 최대 1.0 음수 가능 (최악)
R² · Adjusted R²
R² = 1 - SS_res / SS_tot = 1 - Σ(yᵢ-ŷᵢ)² / Σ(yᵢ-ȳ)²
Adj R² = 1 - (1-R²)×(n-1) / (n-p-1) ← 피처 수 p 보정
R² 해석 기준
0
0.6
0.85
음수: 평균보다 나쁨 0.6~0.85: 실용 범위 0.85+: 우수
R² vs Adjusted R² — 피처 수 영향 분석

from sklearn.datasets import load_diabetes
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split
from sklearn.metrics import r2_score
import numpy as np

X, y = load_diabetes(return_X_y=True)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=.2, random_state=42)

# ─── 피처 수에 따른 R² vs Adj-R² ───
def adj_r2(r2, n, p):
    return 1 - (1 - r2) * (n - 1) / (n - p - 1)

print(f"{'Features':>10} | {'R²':>8} | {'Adj R²':>8}")
print("-" * 34)
for k in range(1, X.shape[1] + 1):
    model = LinearRegression()
    model.fit(X_tr[:, :k], y_tr)
    yp = model.predict(X_te[:, :k])
    r2 = r2_score(y_te, yp)
    ar2 = adj_r2(r2, len(y_te), k)
    print(f"{k:>10} | {r2:>8.4f} | {ar2:>8.4f}")

# ─── 잔차(Residual) 플롯 ───
import matplotlib.pyplot as plt
model_full = LinearRegression().fit(X_tr, y_tr)
yp_full = model_full.predict(X_te)
residuals = y_te - yp_full

fig, axes = plt.subplots(1, 2, figsize=(11, 4))
axes[0].scatter(yp_full, residuals, alpha=.5, color='#7c3aed', s=30)
axes[0].axhline(0, color='k', ls='--', lw=1)
axes[0].set(xlabel='Predicted', ylabel='Residual', title='잔차 플롯')

axes[1].hist(residuals, bins=25, color='#16a34a', alpha=.7, edgecolor='white')
axes[1].set(xlabel='Residual', ylabel='Count', title='잔차 분포')

r2_full = r2_score(y_te, yp_full)
fig.suptitle(f'Linear Regression — R² = {r2_full:.4f}', fontsize=13, fontweight='bold')
plt.tight_layout(); plt.show()
📊 결과 해석
  • 피처 추가 시 R²는 단조 증가하지만 Adj R²는 무의미한 피처에서 감소
  • Diabetes 전체 피처: R² ≈ 0.45 수준 (의료 데이터 특성상 낮음)
  • 잔차가 0 주위에 무작위 분포해야 선형 가정 충족
  • 잔차가 패턴(U자, 팬형)이면 비선형 모델 또는 변환 필요

Silhouette Coefficient
a(i) = 평균 거리(i, 같은 군집 내 다른 점들) ← 낮을수록 좋음
b(i) = min(평균 거리(i, 다른 군집 j의 점들)) ← 높을수록 좋음
s(i) = (b(i) - a(i)) / max(a(i), b(i)) ∈ [-1, 1]

from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score, silhouette_samples
import matplotlib.pyplot as plt
import numpy as np

# ─── 데이터 생성 (진짜 군집 4개) ───
X, y_true = make_blobs(n_samples=400, centers=4,
                        cluster_std=0.8, random_state=42)

# ─── K별 Silhouette Score ───
k_range = range(2, 10)
sil_scores, inertia = [], []
for k in k_range:
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    labels = km.fit_predict(X)
    sil_scores.append(silhouette_score(X, labels))
    inertia.append(km.inertia_)

best_k = k_range.start + np.argmax(sil_scores)
print(f"최적 K: {best_k}, Silhouette Score: {max(sil_scores):.4f}")

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
axes[0].plot(k_range, sil_scores, 'o-', color='#0ea5e9', lw=2)
axes[0].axvline(best_k, color='red', ls='--', lw=1.5, label=f'Best K={best_k}')
axes[0].set(xlabel='K', ylabel='Silhouette Score', title='Silhouette Score by K')
axes[0].legend()

# ─── Silhouette Plot (샘플별) ───
km_best = KMeans(n_clusters=best_k, random_state=42, n_init=10)
labels_best = km_best.fit_predict(X)
sample_sils = silhouette_samples(X, labels_best)
colors = plt.cm.tab10(np.linspace(0, 1, best_k))
y_lower = 10
for i, c in enumerate(colors):
    ith_sils = np.sort(sample_sils[labels_best == i])
    size_i = ith_sils.shape[0]
    y_upper = y_lower + size_i
    axes[1].fill_betweenx(np.arange(y_lower, y_upper),
                           0, ith_sils, facecolor=c, alpha=.8)
    y_lower = y_upper + 10
axes[1].axvline(max(sil_scores), color='red', ls='--', lw=1.5)
axes[1].set(xlabel='Silhouette Coefficient', title=f'Silhouette Plot (K={best_k})')
plt.tight_layout(); plt.show()
📊 결과 해석
  • K=4(실제 군집 수)에서 Silhouette Score 최대 → 정확히 탐지
  • Silhouette Plot에서 폭이 균일하고 평균선(빨간 점선) 오른쪽에 있어야 좋음
  • 평균보다 아래인 샘플이 많으면 해당 군집 재검토
  • Elbow Method(Inertia)와 함께 사용하면 더 신뢰도 높음

from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans
from sklearn.metrics import (silhouette_score,
                              davies_bouldin_score,
                              calinski_harabasz_score)
import pandas as pd

X, _ = make_blobs(n_samples=500, centers=4,
                   cluster_std=0.9, random_state=42)

rows = []
for k in range(2, 9):
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    labels = km.fit_predict(X)
    rows.append({
        'K'         : k,
        'Silhouette': silhouette_score(X, labels),          # ↑ 클수록 좋음
        'DB_Index'  : davies_bouldin_score(X, labels),      # ↓ 작을수록 좋음
        'CH_Index'  : calinski_harabasz_score(X, labels),   # ↑ 클수록 좋음
    })

df = pd.DataFrame(rows).set_index('K').round(4)
print(df)

# 각 지표별 최적 K
print(f"\n[Silhouette 최적 K]: {df['Silhouette'].idxmax()}")
print(f"[DB Index  최적 K]: {df['DB_Index'].idxmin()}")
print(f"[CH Index  최적 K]: {df['CH_Index'].idxmax()}")

  • CH Index는 볼록한(구형) 군집에 유리 → DBSCAN 평가 시 부적합
  • 대용량(n>10만)에서는 Silhouette 대신 CH·DB 우선 사용

from sklearn.datasets import load_breast_cancer
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.model_selection import (cross_validate, StratifiedKFold,
                                      cross_val_score)
import pandas as pd
import numpy as np

X, y = load_breast_cancer(return_X_y=True)

pipe = Pipeline([
    ('sc',  StandardScaler()),
    ('clf', GradientBoostingClassifier(n_estimators=100, random_state=42))
])

# ─── 단일 지표 ───
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(pipe, X, y, cv=cv, scoring='roc_auc', n_jobs=-1)
print(f"ROC-AUC: {scores.mean():.4f} ± {scores.std():.4f}")
print(f"각 fold: {np.round(scores, 4)}")

# ─── 다중 지표 동시 평가 ───
scoring = {
    'accuracy'  : 'accuracy',
    'precision' : 'precision',
    'recall'    : 'recall',
    'f1'        : 'f1',
    'roc_auc'   : 'roc_auc',
}
cv_results = cross_validate(pipe, X, y, cv=cv,
                             scoring=scoring,
                             return_train_score=True,
                             n_jobs=-1)

rows = []
for metric in scoring:
    test_key  = f'test_{metric}'
    train_key = f'train_{metric}'
    rows.append({
        '지표'       : metric,
        'Train Mean' : cv_results[train_key].mean(),
        'Test Mean'  : cv_results[test_key].mean(),
        'Test Std'   : cv_results[test_key].std(),
        'Overfitting': cv_results[train_key].mean() - cv_results[test_key].mean(),
    })

df = pd.DataFrame(rows).set_index('지표').round(4)
print("\n[다중 지표 교차검증 결과]\n", df.to_string())

# ─── 과적합 진단 ───
overfit = df['Overfitting']
print(f"\n[과적합 판단] Train-Test 차이가 0.05 이상인 지표:")
print(overfit[overfit > 0.05])
📊 결과 해석
  • cross_validate로 여러 지표를 한 번에 계산 → 학습 시간 절약
  • Train-Test 차이(Overfitting 열) > 0.05이면 과적합 의심
  • Test Std가 클수록 모델이 fold에 민감 → 더 많은 fold 또는 앙상블 필요
  • n_jobs=-1: 병렬 처리로 속도 향상
💡 팁: 시계열 데이터 → TimeSeriesSplit 필수. KFold 사용 시 미래 데이터가 훈련에 포함되는 누출 발생!

import numpy as np
import pandas as pd
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import cross_validate, StratifiedKFold, KFold

# ════════════════════════════════════════
# ① 분류 모델 종합 평가 함수
# ════════════════════════════════════════
def evaluate_classifier(model, X, y, cv=5, pos_label=1):
    """분류 모델을 다중 지표로 교차검증 평가"""
    from sklearn.metrics import make_scorer, f1_score, roc_auc_score

    scoring = {
        'accuracy'  : 'accuracy',
        'precision' : 'precision_weighted',
        'recall'    : 'recall_weighted',
        'f1'        : 'f1_weighted',
        'roc_auc'   : 'roc_auc',
    }
    skf = StratifiedKFold(n_splits=cv, shuffle=True, random_state=42)
    results = cross_validate(model, X, y, cv=skf, scoring=scoring,
                              return_train_score=True, n_jobs=-1)
    rows = {}
    for metric in scoring:
        test_s  = results[f'test_{metric}']
        train_s = results[f'train_{metric}']
        rows[metric] = {
            'Train': f"{train_s.mean():.4f}",
            'Test' : f"{test_s.mean():.4f} ± {test_s.std():.4f}",
            'Gap'  : f"{train_s.mean()-test_s.mean():.4f}",
        }
    return pd.DataFrame(rows).T

# ════════════════════════════════════════
# ② 회귀 모델 종합 평가 함수
# ════════════════════════════════════════
def evaluate_regressor(model, X, y, cv=5):
    """회귀 모델을 다중 지표로 교차검증 평가"""
    scoring = {
        'MAE' : 'neg_mean_absolute_error',
        'RMSE': 'neg_root_mean_squared_error',
        'R2'  : 'r2',
    }
    kf = KFold(n_splits=cv, shuffle=True, random_state=42)
    results = cross_validate(model, X, y, cv=kf, scoring=scoring,
                              return_train_score=True, n_jobs=-1)
    rows = {}
    for metric, key in scoring.items():
        test_s  = -results[f'test_{key}'] if 'neg' in key else results[f'test_{key}']
        train_s = -results[f'train_{key}'] if 'neg' in key else results[f'train_{key}']
        rows[metric] = {
            'Train': f"{train_s.mean():.4f}",
            'Test' : f"{test_s.mean():.4f} ± {test_s.std():.4f}",
        }
    return pd.DataFrame(rows).T

# ════════════════════════════════════════
# ③ 사용 예시
# ════════════════════════════════════════
from sklearn.datasets import load_breast_cancer, load_diabetes
from sklearn.ensemble import GradientBoostingClassifier, GradientBoostingRegressor

# 분류
X_c, y_c = load_breast_cancer(return_X_y=True)
clf_pipe = Pipeline([('sc', StandardScaler()),
                     ('clf', GradientBoostingClassifier(random_state=42))])
print("=== 분류 평가 ===")
print(evaluate_classifier(clf_pipe, X_c, y_c).to_string())

# 회귀
X_r, y_r = load_diabetes(return_X_y=True)
reg_pipe = Pipeline([('sc', StandardScaler()),
                     ('reg', GradientBoostingRegressor(random_state=42))])
print("\n=== 회귀 평가 ===")
print(evaluate_regressor(reg_pipe, X_r, y_r).to_string())
📊 올인원 템플릿 포인트

  • neg_ prefix 지표는 부호 반전으로 직관적 표시
  • Train vs Test Gap으로 과적합을 한눈에 감지
  • 함수 재사용으로 모델 비교 시 반복 코드 제거
