KV Cache 시각화

LLM이 토큰을 생성할 때 이전에 계산한 Key · Value를 재사용해 중복 연산을 제거하는 기법입니다.

Attention = softmax( Q · Kᵀ / √d ) · V

새 토큰의 Q는 매번 바뀌지만, 이전 토큰들의 K · V는 변하지 않습니다.

핵심 개념 정리

KV Cache 없음

N번째 토큰 생성 시, 이전 토큰 1~N 모두의 K·V를 다시 계산합니다. 시퀀스가 길어질수록 연산이 O(N²)으로 증가합니다.

KV Cache 있음

이전 토큰들의 K·V는 메모리에서 읽어오고, 새 토큰의 K·V만 계산합니다. 연산이 O(N)으로 줄어듭니다.

트레이드오프

연산은 줄지만 K·V를 GPU VRAM에 저장해야 합니다. 128K 컨텍스트에서는 캐시 자체가 수십 GB를 차지하기도 합니다.

관련 기술

VRAM 절감을 위해 Paged Attention, Multi-Query Attention, Sliding Window 등 다양한 최적화 기법이 활용됩니다.
